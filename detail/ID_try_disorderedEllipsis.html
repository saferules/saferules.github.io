捕获所有异常的 catch(...) 子句应位于最后

9.6.4 ID_try_disorderedEllipsis
目录 › next › previous

如果 catch(...) 子句排在前面，其后面的 catch 子句会失去作用。

示例：

try {
    ....
} catch (...) {   // Non-compliant, disordered catch-all handler
    ....
} catch (E&) {   // Unreachable
    ....
}

应改为：

try {
    ....
} catch (E&) {
    ....
} catch (...) {   // Compliant
    ....
}
