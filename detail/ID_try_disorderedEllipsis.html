捕获所有异常的 catch(...) 子句应位于最后

ID_try_disorderedEllipsis
Home › next › previous
如果 catch(...) 子句排在前面，其后面的 catch 子句会失去作用。 示例：
try { .... } catch (...) { // Non-compliant, disordered catch-all handler .... } catch (E&) { // Unreachable .... }
应改为：
try { .... } catch (E&) { .... } catch (...) { // Compliant .... }
