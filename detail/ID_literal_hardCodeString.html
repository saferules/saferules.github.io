在字符串常量中用转义字符表示制表符和控制字符

ID_literal_hardCodeString

未经转义的制表符和控制字符不利于阅读和维护。 C++ 原始字符串（raw string）不受本规则限制，但不建议在原始字符串中使用制表符和换行符之外的控制字符。 示例：
const char* s = " "; // Non-compliant
例中字符串包含制表符，易被误解为空格，在终端上复制粘贴后也可能变成数目不确定的空格，导致与预期不符的结果。 故应使用转义字符：
const char* s = "\t"; // Compliant
