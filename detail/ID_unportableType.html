用 stdint.h 中的类型代替 short、int、long 等类型

ID_unportableType
Home › next › previous
short、int、long 等类型的取值范围由执行环境的架构决定，可移植性较差。 示例：
struct T { long int x; // Non-compliant long long y; // Non-compliant short z; // Non-compliant };
例中成员变量在不同的平台会有不同的取值范围，C99 引入 stdint.h 解决了这一问题。 应改为：
#include <stdint.h> // Or <cstdint> in C++ struct T { int32_t x; // Compliant int64_t y; // Compliant int16_t z; // Compliant };
另外，char、wchar_t 等基本类型均有此问题，在有高可移植性要求的代码中应避免直接使用基本类型。

依据

ISO/IEC 9899:2011 6.2.5(5) ISO/IEC 9899:2011 7.20 ISO/IEC 14882:2011 3.9.1(2)
