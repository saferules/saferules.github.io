遵循统一的缩进风格

17.5 ID_inconsistentIndent
目录 › next › previous

缩进方式不统一会使代码结构混乱，严重降低可读性，也易导致逻辑错误。

缩进应清晰地体现语句的从属关系，函数、分枝、循环等代码块的内容应相对代码块的起始位置缩进一个层级，建议使用 2、4 或 8 个空格作为一个缩进层级，并避免使用制表符，以便代码在不同的编辑器中可以保持一致的样式。

示例：

switch (a) {
case 0:
    switch (b) {
case 1:           // Inconsistent
    break;        // Inconsistent
    }
    break;
default:
    break;
}
