捕获异常后不应直接再次抛出异常

7.19 ID_catch_justRethrow
目录 › next › previous

捕获异常后将其直接再次抛出是没有意义的，还会造成不必要的开销。

示例：

void foo() {
    try {
        bar();
    }
    catch (...) {
        throw;      // Non-compliant
    }
}

例中的 catch 子句是没有意义的，应将其去掉或对异常进行有效处理。
