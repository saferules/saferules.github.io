没有 else 子句的 if 语句与其后续代码相同是可疑的

9.1.7 ID_if_identicalSucceedingBlock
目录 › next › previous

if 语句与其同一作用域的后续代码完全相同是可疑的，可能是由复制粘贴造成的错误。

示例：

if (condition) {
    a = SOME_VALUE;
}
a = SOME_VALUE;  // Rather suspicious

例中对变量 a 的赋值是没有意义的。
