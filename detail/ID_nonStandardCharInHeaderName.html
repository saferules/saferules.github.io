include 指令中禁用不合规的字符

ID_nonStandardCharInHeaderName
Home › next › previous
字母、数字、下划线、点号之外的字符可能与文件系统存在冲突，也可能导致标准未定义的行为，不应出现在头文件和相关目录名称中。 示例：
#include <"foo"> // Non-compliant #include <foo*> // Non-compliant #include <foo'bar> // Non-compliant #include <foo> // Compliant #include <foo.h> // Compliant #include <foo_bar> // Compliant
可以用 / 作为路径分隔符，但不应出现 // 或 /*， 如：
#include <foo//bar.h> // Non-Compliant, undefined behavior #include <foo/*bar.h> // Non-Compliant, undefined behavior
另外，某些平台的文件路径不区分大小写，建议在头文件名称中只使用小写字母以提高可移植性。
