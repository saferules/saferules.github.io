不应存在多余的逻辑子表达式

ID_redundantCondition
Home › next › previous

逻辑或、逻辑与的子表达式有包含关系时，其中的一个表达式是多余的。

示例：

a > b && a != b // None-compliant, a != b is redundant a < b || a != b // None-compliant, a < b is redundant a >= b && a == b // None-compliant, a >= b is redundant a == b || a <= b // None-compliant, a == b is redundant

多余的子表达式很可能意味着某种错误，需认真对待。
